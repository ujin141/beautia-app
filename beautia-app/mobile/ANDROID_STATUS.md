# Android 에뮬레이터 상태

## ✅ 현재 상태

- **에뮬레이터**: Pixel_5 (emulator-5554)
- **상태**: 실행 중 및 준비 완료
- **앱 설치**: 진행 중

## 📱 다음 단계

1. **에뮬레이터 화면 확인**
   - Pixel_5 에뮬레이터 창이 열려 있어야 합니다
   - Android 홈 화면이 표시되어야 합니다

2. **앱 자동 설치**
   - `npm run android` 명령어가 실행 중입니다
   - 앱이 자동으로 에뮬레이터에 설치되고 실행됩니다

3. **앱 실행 확인**
   - 에뮬레이터에서 BEAUTIA 앱이 자동으로 열립니다
   - 로그인 화면이 표시되어야 합니다

## 🔧 문제 해결

### 앱이 설치되지 않는 경우

1. **에뮬레이터 부팅 완료 확인**
   - 에뮬레이터의 Android 홈 화면이 완전히 로드되어야 합니다
   - 잠금 화면이 표시되면 잠금을 해제하세요

2. **수동 설치**
   ```bash
   cd mobile
   npx expo start --android
   ```
   터미널에서 `a` 키 누르기

3. **에뮬레이터 재시작**
   - 에뮬레이터 종료 후 다시 시작
   ```powershell
   .\start-android.ps1
   ```

### API 연결 실패

- `mobile/app.json`의 `apiBaseUrl`이 `http://10.0.2.2:3000`로 설정되어 있는지 확인
- 백엔드 서버가 `http://localhost:3000`에서 실행 중인지 확인

## 📝 참고사항

- **API URL**: Android 에뮬레이터에서는 `localhost` 대신 `10.0.2.2`를 사용합니다
- **백엔드 서버**: `http://localhost:3000`에서 실행되어야 합니다
- **핫 리로드**: 코드를 저장하면 자동으로 앱이 리로드됩니다

## 🎯 성공 확인

앱이 정상적으로 실행되면:
- ✅ 로그인 화면 표시
- ✅ 에뮬레이터에서 앱 인터랙션 가능
- ✅ API 연결 성공 (로그인 시도 시)
